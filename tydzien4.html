GIT

git status | git add . | git commit -m"opis zmiany" | git push - to podstawowe najczęściej używane komendy

Repozytorium - projekt, jego wszystkie pliki w różnych etapach.
Commit - obecny stan projektu i opis, co się zmieniło od ostatniej wersji, rodzaj "save-a".

  1. Instalujemy GIT-a: https://git-scm.com/
  2. wchodzimy do terminala windows/linux/mac -można użyć Git Bash, króry na windowsie zainstalowal sie z GIT’em, dzialają w nim polecenia z terminala windows jak i linux.
  3. Wpisz polecenia:
    • git config --global user.name "Jan Nowak" [tworzy konto Jan Nowak]
    • git config --global user.email jannowak@example.com [dodaje email]
    • git config -- get user.name [pobiera dane wlasciciela konta]
    • git config -- get user.email [pobiera email wlasciciela]
    • git --version - pokazuje wersję zainstalowanego Gita
  4. Wchodzisz do katalogu z projektem (cały czas w terminalu) i wprowadzasz polecenia:
    • git init - zakładanie pustego repozytorium - koniecznie w katalogu z plikami projektu
    • git status - pokazuje aktualny stan repozytorium, nic nie zmienia. Na czerwono wyswietli pliki, które nie sa jeszcze zapisane na repozytorium, lub których dane zmienily sie od ostatniego commita. Na zielono wyswietli pliki zacommitowane lub te, których zawartosc nie zmienila sie od ostatniego commita.
    • git add . - (git add ‘spacja’ KROPKA) dodaje WSZYSTKIE pliki z bieżącego katalogu i podkatalogów do zarejestrowania ("zacommitowania")
    • git commit -m "OPIS CO SIE ZMIENILO od ostatniego razu, koniecznie w j. ang." - dodawanie nowego lub kolejnego rejestru/commitu (w terminalu)
    • git commit --amend -m "New description" - zmienia opis ostatniego commitu
    • gitk - dodatek pokazujący historię naszego projektu i zmiany w formie graficznej
    • clear - czyści terminal
  5. W VisualStudioCode też mamy wbudowany GIT. Mozesz tez zainstalowac w VSC dodatek GIT LENS. Z pozycji VSC można rownież commitowac, ale na początek aby nabrać wprawy i wiedzy jak co dziala polecam korzystać z terminala.

Życie w pudełku

Poruszane tematy:

display:block / inline
overflow: hidden / visible / auto

display:- może mieć różne wartości (teraz poznamy tylko 3 z nich)

  • display:block;- kontener zajmuje całą szerokość strony, domyślny dla wielu znaczników. Kontenery z właściwością display: block; układają się na stronie jeden pod drugim
  • display: inline; - kontener zajmuje dokładnie tyle szerokości i wysokości ile znajdująca się w nim treść. Kontenery z właściwością display:inline; układają się na stronie obok siebie (w linii) aż zapełnią całą szerokość ekranu, następnie przechodzą do kolejnej linii. Dla display;inline; - nie można nadawać wartości width i height.
  • display: inline-block; - wyświetla się jak inline, ale może otrzymać właściwości jak kontener z własnością block, np. width i height.

overflow - może mieć wartości:

  • overflow: hidden; - jeśli treść kontenera zawiera większy obszar niż rozmiar kontenera (w pionie lub w poziomie) to “nadmiarowej” treści nie będzie widać
  • overflow: visible; - jeśli treść kontenera zawiera większy obszar niż rozmiar kontenera (w pionie lub w poziomie) to “nadmiarową” treść będzie widać, będzie ona wychodziła poza rozmiar kontenera i będzie nachodzić na elementy znajdujące się obok lub pod naszym kontenerem.
  • overflow:auto; - przy nadmiarze treści pojawia się scroll w pionie lub poziomie.

JEDNOSTKI:

  • px - piksele
  • % procenty (np width: 50%; jest to 50% szerokości rodzica)

Elementy w systemie box-model na stronie są prostokątami, które “obrabiamy” aby uzyskać odpowiedni kształt.

Jeśli ustawiamy wielkość elementu, mamy dwie możliwości:

  • box-sizing (domyślna) - podany wymiar ustawia dla content box
  • border-box - ustawia narzucony wymiar width i height dla wszystkiego włącznie z border, tylko marginesu nie uwzględnia.

Budowa każdego prostokąta:

  • content - to treść znajdująca się w naszym elemencie
  • padding - to wewnętrzny margines. Domyślnie wynosi zero
  • border - to obramowanie naszego elementu, domyślnie border wynosi zero.
  • margin - ustawia margines od innych elementów, czyli odległość w jakiej obramowanie naszego elementu znajduje się od innego elementu.
    margin: auto; - wyśrodkowuje elementy o właściwości display:block; ele pod warunkiem, że elementy mają zdefiniowaną wartość width. (Jeśli element nie ma zdefiniowanego width, to zajmuje całą szerokość rodzica.)
